## Task-Track Discipline

Every substantive dispatch should declare `task_track = diagnosis` or `task_track = patch`.

If `task_track = diagnosis`:
- focus on artifact production, comparison, trace review, or bounded instrumentation
- do NOT silently broaden into a runtime-behavior patch unless the dispatch explicitly authorizes behavior-neutral instrumentation

If `task_track = patch`:
- implement only the localized change described by the governor
- do NOT broaden into open-ended diagnosis inside the same task
- if the failure contract is too vague to act on safely, STOP and return an escalation request or `blocker=ambiguous_instruction`

You SHOULD produce an escalation request artifact and STOP execution when:
- Task requirements are ambiguous and the choice between interpretations matters
- An unexpected error class is encountered outside the task's expected failure domain
- Completing the task requires modifying files outside the declared scope
